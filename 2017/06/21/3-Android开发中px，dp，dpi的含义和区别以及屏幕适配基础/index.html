Android开发中px，dp，dpi的含义和区别以及屏幕适配基础

发表于 2017-06-21

看过很多类似这样的文章去分享这三者的区别，今天我也在这里谈下我的理解。在了解这三个单位之前，首先了解下什么是像素（px）。

什么是像素

我们经常看到过一些大型的舞蹈，很多人排列整齐，有序的站着，每个人手中都拿着一个带颜色的扇子或者气球，这些演员可以通过手中的不同颜色的气球，共同组成一个文字或者图像。这里每个气球其实就是一个像素（px），一次可以展示一种颜色。所以拿气球的人越多，可以组成的图像和文字就越逼真，也即像素越多，越能展示更多的细节，图像显示的质量越高。我们平常看到的手机的参数比如1280 x 720 ，意思就是长宽分别有1280个和720个像素点，每个像素点负责展示一个颜色。像素个数在一定程度上可以反映手机显示的质量，为什么只是一定程度上，而不是决定性的作用呢？因为显示的质量不仅仅和像素个数有关系，还和dpi有关系。

PPI或DPI

ppi也就是每英寸像素（Pixels Per Inch）个数，又称为dpi（Dots Per Inch），他是由屏幕对角线的像素点的个数除以屏幕对角线的长度得到的。也就是每英寸的像素个数越多 ，dpi越高。dpi对于屏幕的显示的质量是有很大关系的，往往dpi越高，每英寸展示出的图像细节越多，显示质量越高。

比如屏幕对角线的像素个数是880个像素点，屏幕对角线尺寸是5.5英寸，所以这个屏幕的dpi是880 ÷ 5.5 = 160dpi；如果对角线的像素个数是1320个，那么这个屏幕的dpi是1320 ÷ 5.5 = 240dpi，很显然240dpi的屏幕要比160dpi的屏幕显示的质量要好。因为它在每个英寸的范围内像素个数更多，能展示更多的细节。所以看一个手机的显示质量不仅仅看像素个数，还要参照ppi（dpi）。

DP

我们都知道，目前Android手机的碎片化很严重，屏幕的种类非常多。如果我们在开发的过程中用像素的个数来表示控件的高度，比如高度是200px，那么发生的情况就是这个控件在有的手机上可能占用半屏的空间，在有的手机上占用屏幕1/3的空间，这显然是不可行的；用英寸表示更不行。所以需要有一个单位，这个单位可以表示一个控件的尺寸，让这个控件的显示效果在不同的手机上显示的效果接近一致，这就是dp或者 dip (Density-Independent pixel)。

dp是一个在开发过程中经常用到的单位，可以使得开发者设置的长度能够根据不同屏幕dpi获得不同的像素(px)数量，从而接近统一的显示效果。Google规定：如果将一个控件的高度设置为1dp，那么在160dpi的屏幕下，这个控件的高度用1px表示。由此可以得出在240dpi下，1dp为 1 x （240÷160) = 1.5px，2dp为2 x （240÷160） = 3px，即在240dpi的屏幕下，2dp用3个像素表示。

所以用m dpi的屏幕表示 a dp用的像素个数是 a x （m÷160）；

DP和英寸的关系

如果我们想在不同的dpi屏幕的手机表示160dp，那么都分别用到多少像素表示呢？看下表：

分辨率 每英寸像素个数 要表示的dp值 要表示的dp值对应的像素个数

320 x 480 160dpi 160dp 160dp x 160dpi/160dpi = 160px
480 x 800 240dpi 160dp 160dp x 240dpi/160dpi = 240px
1080 x 1920 360dpi 160dp 160dp x 360dpi/160dpi = 360px

可以看到，在不同的dpi屏幕的手机上表示160dp，所用的像素个数和dpi相同即和每英寸用到的像素个数相同，所以得出的结论是：无论任何dpi的屏幕，表示160dp都是一英寸的长度，即：一英寸是160dp。

屏幕适配

（1）图标大小按比例缩放制作。由上面的分析，可以知道如果将一个控件的高度设置为1dp，那么：
在120dpi下，1dp用120÷160 = 0.75px表示；
在160dpi下，1dp用160÷160 = 1px表示；
在240dpi下，1dp用240÷160 = 1.5px表示；
在320dpi下，1dp用320÷160 = 2px表示。
而一个px的长度是确定的，所以同样表示1dp的长度，不同dpi屏幕上用的长度也是不一样的，这个长度的比例和用到的像素个数有关。

即在120dpi、160dpi、240dpi、320dpi，需要用到的图标元素的尺寸比例为 0.75 : 1 : 1.5 : 2。按照这个比例去分别做图标，可以在大部分的手机上有接近一致的效果。

（2）在开发过程中，除文字控件外，尽量使用dp来表示控件的尺寸。
