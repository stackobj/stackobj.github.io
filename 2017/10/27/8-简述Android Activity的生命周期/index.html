简述Android Activity的生命周期

发表于 2017-10-27

Activity的生命周期是Android最基本和最重要的知识点之一。如果能深入地理解Activity的生命周期，对我们出开发优秀的APP很有帮助，这篇文章就来讲讲这个知识点。

一、生命周期

Activity是一个View对象的容器，也是View的事件分发机制的起点。它可以构成APP一个个可见的界面。官方给出Activity的生命周期图是这样的：

Activity从创建，到被Android回收，Android系统会依次回调以下几个方法：

1、onCreate()

Activity正在被创建，这个时候Activity的View是不可见的，在这里我们可以进行一些初始化的工作，如setContentView加载Activity的内容View，或者对控件和变量进行初始化。

2、onStart()

回调这个方法的时候，Activity已经可见了，但是此时还不能和用户交互。从生命周期的流程图我么可以看到，每次Activity被切换为可见状态的时候，都会执行onStart，所以这里不便于进行控件和变量的初始化。

3、onResume()  

回调这个方法的时候，Activity是可见的，并且可以和用户交互。

4、onPause()

表示Activity进入暂停状态，Activity正在停止，此时Activity是可见的，我们可以在这里进行一些简单的数据存储或者初始化一些变量。

5、onStop()

表示Activity进入停止状态，此时Activity是不可见的。这个时候Activity对象在内存中。

6、onDestory()

Activity被销毁以后调用，此时Activity不可见，并且Activity已经不在内存中了。这里我们释放一些资源。

7、onRestart()

当用户从桌面切换为Activity，或者从后一个Activity切换回来的时候，会出发这个方法。这里一般不做什么操作。

当进行以下几种情况时的回调方法执行顺序：

① Activity正常启动：onCreate()->onStart()->onResume()；
② Activity正常退出：onPause()->onStop()->onDestory()；
③ 一个Activity启动另一个Activity: onPause()->onStop(), 再返回：onRestart()->onStart()->onResume()；
④ Activity横竖屏切换：onPause()->onStop()->onDestory()->onCreate()->onStart()->onResume()
⑤ 按back键退出： onPause()->onStop()->onDestory()，再进入：onCreate()->onStart()->onResume()；
⑥ 按home键退出： onPause()->onStop()，再进入：onRestart()->onStart()->onResume()；

二、onSaveInstanceState() & onRestoreInstanceState()

1、当某个Activity变得容易被系统销毁，而非用户主动销毁时，则会在onPause()或onStop()方法之前回调onSaveInstanceState()，例如：

① 当用户按下HOME键时。因为当用户按下HOME键，那么这个Activity所在的栈被放入后台，这个后台的栈里的Activity可能会因为内存资源紧张而被回收，所以会执行onSaveInstanceState()，让用户有机会在这里保存界面的状态，如输入框的内容。

② 长按HOME键，选择运行其他的程序时。
③ 按下电源按键（关闭屏幕显示）时。
④ 从activity A中启动一个新的activity时。
⑤ 屏幕方向切换时，例如从竖屏切换到横屏时

示例代码如下：

protected void onSaveInstanceState(Bundle outState){ 
       super.onSaveInstanceState(outState); 
       outState.putInt("key1", value1); 
       outState.putString("key2", value2); 
}

3、当某个Activity不是用户主动销毁，并且又返回此Activity的时候，onRestoreInstanceState()会被调用。当要恢复状态的时候可以在onCreate里面执行恢复，也可以在OnRestoreInstanceState执行恢复。

示例代码如下：

protected void onCreate(Bundle bundle) { 
       super.onCreate(bundle); 
       if (bundle != null) { 
       } 
}

或：

protected void onRestoreInstanceState(BundlesavedInstanceState) { 
   super.OnRestoreInstanceState(savedInstanceState); 
   value1 = savedInstanceState.GetInt("key1", value1_dafault); 
   value2 = savedInstanceState.GetString("key2", "value2_default"); 
}

另外如果要保存状态，其实也可以在onPause里面用SharedPreference来保存，在onCreate里面恢复状态。
